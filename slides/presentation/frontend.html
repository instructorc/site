A few HTML Considerations!

Indentation + Nesting of HTML tags
Indentation is good coding practice - Easier to read and understand code. Nesting the tags establishes a child/parent relationship and complies to DOM tree structure

Understanding the <div> tag,
The div tag serves as a container to group tags. Other container tags include <header>,<nav>, <footer>, <section>, <article>,<aside>.

Attributes
Placed in the opening tag and provides additional information to tag.

How do we connect CSS to our HTML?

Internal
uses the style tag within head of your site and applies CSS to an HTML document.
<style> </style>

External
uses the link tag to link multiple HTML documents to CSS file. External style sheets are separate documents that have a .css extension
<link href="style.css" rel="stylesheet" type="text/css" />

Inline
uses the style attribute and is tag specific and applies CSS to a tag.
<div style="color: red;">

Type Selector
Simply use the name of HTML tag.
HTML: <button>Save + Print List</button>

id Selector
Used to target a specific HTML tag.
HTML: <div id="goat"> Jordan </div>

class Selector
Can be target multiple HTML tags.
HTML:
<div class="team"> Pippen </div>
<div class="team"> Rodman </div>
<div class="team"> Harper </div>

Desendent Selector
Targeting child tag of a parent.
HTML:
<aside>
    <p>Greatest team! </p>
</aside>

Getting Started with Page Layout!

Layout Strategy

A number of strategies can be used to design the layout of your page. You can use CSS flexbox, CSS grid, float, or a mixture of the three.

Traditional Float

Using float, clear, position

CSS Flexbox

Lends itself to responsive design. An easier way to work with pages.

CSS Grid

Implementation is similar to CSS Flexbox. CSS Grid defines a grid where you can place box elements on to the grid.

CSS Flex Steps

Step 1
Define a container element as a grid with display: flex;

Step 2
Give structure to flex-items and set the flex-grow, flex-shrink, and flex-basis for each box. flex: 1 0 300px;

Step 3
Define flow of boxes using either flex-direction or flex-wrap flex-wrap: wrap;

Chris Coyier's CSS tricks tutorial on using CSS Flexbox

Our Layout Strategy

CSS Grid

CSS Grid Steps

Step 1
Define a container element as a grid with display: grid

Step 2
Set the column and row sizes with grid-template-columns and grid-template-rows

Step 3
Place child elements into the grid with grid-column and grid-row

Chris Coyier's CSS tricks tutorial on using CSS Grid

Defining the Grid

								
									.grid {
									 display: grid;
									 grid-template-columns: repeat(4, 1fr);
									 grid-template-rows: repeat(4, 75px);
									}

Defining a Complex Grid

								
									.grid {
									 display: grid;
									 grid-template-columns: repeat(4, 1fr);
									 grid-template-rows: 40px 2fr repeat(2, minmax(75px, 1fr));
									}

Positioning items in the Grid using line numbers

								
									.grid {
									 	display: grid;
									 	grid-template-columns: repeat(4, 1fr);
									 	grid-template-rows: 40px 2fr repeat(2, minmax(75px, 1fr));
									}
									
									#item1{
										grid-column-start: 2;
										grid-column-end: 4;
										grid-row-start: 3;
										grid-row-end: -1;
										background-color: red;
									}
									
									#item2{
										grid-column: 3;
										grid-row: 1;
										background-color: red;
									}

Positioning items in the Grid using area names

								
									.grid {
									 	display: grid;
									 	grid-template-columns: repeat(4, 1fr);
									 	grid-template-rows: 40px 2fr repeat(2, minmax(75px, 1fr));
									 	grid-template-areas: ". . . ." 
                                                             "item1 . . ." 
                                                             "item1 item2 item2 item2" 
                                                             " . item2 item2 item2";
									}
									
									#item1{
										grid-area: item1;
										background-color: red;
									}
									
									#item2{
										grid-area: item2;
										background-color: red;
									}

Accessible and Responsive web design

Validating your code

  • Valid code conforms to W3C rules.

  • Valid code enhances browser compatibility, accessibility, and exchange of data
  • The most common mistakes include:
    • No doctype declaration
    • Missing closing tags
    • Missing alt attributes in image elements
    • Incorrect tag nesting
    • Unquoted attributes
  • W3C Validation Service Link

Accessibility Principles

Perceivable
People can see the content or hear it

Operable
people can typing or by voice

Understandable information
people get clear and simple language

Robust content
people can use different assistive technology
