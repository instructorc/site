.NET MAUI Architecture

Mobile App Development using .NET MAUI

XAML
a language that uses XML syntax and enables controls to be added to a user interface in a declarative, hierarchical way.

You can add controls in the form of XML elements, and specify control properties with XML attributes.

XAML is designed with today's powerful graphic's cards in mind

Seperation of Concern
XAML can be rendered without any code. The interaction logic (C# code) is placed in a different file than you place the GUI code.

The UI is not generated using c# code but rather XML. The UI can developed totally separate for the c# code.

Code-Behind Files
C# files that are dynamically linked to the XAML files.

The application's logic layer

Model-View-ViewModel (MVVM) design pattern

Model-View-ViewModel (MVVM) is a structural design pattern that separates objects into three distinct groups:

  • Models hold application data. They’re usually structs or simple classes.
  • Views display visual elements and controls on the screen. They’re typically subclasses of UIView.
  • View models transform model information into values that can be displayed on a view. They’re usually classes, so they can be passed around as references.

  • Object Element Syntax
    • Begins with a left angle bracket (<), followed immediately by the type name of the class or structure being instantiated. Zero or more spaces can follow the type name, and zero or more attributes may also be declared on the object element, with one or more spaces separating each attribute pair.
  • Property Element Syntax
    • Instead of the property being specified as an attribute within the element tag, the property is specified using an opening element tag in elementTypeName.propertyName form, the value of the property is specified within, and then the property element is closed.

.NET MAUI Controls and Properties

Types of Controls

Pages
A page usually occupies all of the screen, or window, and each page typically contains at least one layout

Layout
Assist with the layout of the items on the a page

Views
Ex. views are the UI objects such as labels, buttons, and sliders that are commonly known as controls or widgets in other environments

Pages - Help you organize the GUI

  • ContentPage - displays a single view, and is the most common page type
  • FlyoutPage - manages two related pages of information – a flyout page that presents items, and a detail page that presents details about items on the flyout page
  • NavigationPage - provides a hierarchical navigation experience where you're able to navigate through pages, forwards and backwards, as desired
  • TabbedPage - consists of a series of pages that are navigable by tabs across the top or bottom of the page, with each tab loading the page content.

Layout Panels - Help you organize the GUI

  • StackLayout - organizes elements in a one-dimensional stack, either horizontally or vertically. The Orientation property specifies the direction of the elements, and the default orientation is Vertical.
  • AbsoluteLayout - used to position and size elements using explicit values, or values relative to the size of the layout.
  • Grid - is used for displaying elements in rows and columns, which can have proportional or absolute sizes. A grid's rows and columns are specified with the RowDefinitions and ColumnDefinitions properties
  • FlexLayout - displays child elements either horizontally or vertically in a stack.
    A FlexLayout can also wrap its children if there are too many to fit in a single row or column

Data Binding

What is Data Binding?

Data binding allows you to connect properties of controls to the value of other controls. You can also define resources and use code defined in classes outside your views as a data source for both values of properties and as content for controls.

4 - Components of Data Binding

Data binding is a way of declaratively connecting controls with data

Binding Target
specifies the object on which the binding is used.

Target Property
specifies the property to set

Binding Source
specifies the object used by the binding

Source Property
specifies which property holds the data

							

<Label x:Name="output_lb" Text="Hello!!" 
		  BindingContext="{x:Reference Name=submit_btn}" 
          TextColor="{Binding Path=TextColor}"/>

<Button x:Name="submit_btn" Text="Click Here" TextColor="Orange"/>

namespace mplsrenters_mobile_app;

public partial class EnterClient : ContentPage
{
	Person person = new Person { Name = "Salman", Age = 26 };
	public EnterClient()
	{
		InitializeComponent();
		this.BindingContext = person;
	}
	private async void Button_Click(object sender, EventArgs e)
	{
		string message = person.Name + " is " + person.Age;
		
		await DisplayAlert("Info.", message, "OK");
	}

}

public class Person
{

	private string nameValue;

	public string Name
	{
		get { return nameValue; }
		set { nameValue = value; }
	}

	private double ageValue;

	public double Age
	{
		get { return ageValue; }

		set
		{
			if (value != ageValue)
			{
				ageValue = value;
			}
		}
	}

}

Understanding Delegates

A delegate is a type that represents references to methods with a particular parameter list and return type.

  • Delegates are used to pass methods as arguments to other methods. Can be used to implement callback functionality
  • Process - Declare a delegate | Set a target method | Invoke a delegate
  • There are two types of delegates available - Single Cast Delegate and Multi Cast Delegate
  • A good reference on understanding delegates
							
using System;  
using System.Collections.Generic;  
using System.Linq;  
using System.Text;  
using System.Threading.Tasks;  
namespace DelegatesDemo {  
    class Program {  
        delegate void X(string a); //declaring a delegate
        static void Display(string I) {  //Target method
            Console.WriteLine("My Name is :" + I);  
        }  

        static void Main(string[] args) {  
            X objD = new X(Display); //Instantiating the delegate 
            objD("Instructor Chris");  //Invoking the delegate
            Console.Read();  
        }  
    }  
}
Delegates are representatives. Think about our election process. Each state has a certain number of delegates that will support a particular candidate. The same thing occurs in programming where we have delegate functions and they ultimately represent a function that has been defined somewhere in our code. Delegate functions provide a way of representing methods with the same signature that can perform and operate in different ways. When doing event driven program, most people use delegate functions without even knowing they are using theses particular functions. In simple terms a delegate allows you to pass a function/method as an arguement to another method.
