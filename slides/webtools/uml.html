What is UML

UML

A family of graphical notations to describe and design software systems, especially those using an object-oriented approach.

UML Survey

Grady Booch, one of the most important developer of Unified Modeling Language, stated that "For 80% of all software only 20% of UML is needed"

Most Commonly Used Diagrams

Use Case Diagram
behavior diagram used to describe a set of actions (use cases) that some system(s)(subject) should or can perform in collaboration with one or more external users of the system (actors).

Activity Diagram
portrays the control flow from a start point to a finish point showing the various decision paths that exist while the activity is being executed.

Sequence Diagram
depicts interaction between objects in a sequential order i.e. the order in which these interactions take place

Class Diagram
Class diagrams model the type of objects in the system, along with the relationships among them. Class diagrams show a static view of system

Communication Diagram
a kind of interaction diagram, shows how objects interact

Classifiers & Features

A classifier is a category of UML elements that have some common features, such as attributes or methods. In class diagrams, the two most common classifiers are regular classes and interfaces

  • Class diagram components
    • Class or Interface
    • Structural and Behavior Features which translates to attributes and methods
    • Visibility - Private | Protected | Public
    • Static and Instance Scope

Relationships

Graphically, a relationship is rendered as a path, with different kinds of lines used to distinguish the different relationships.

Relationship Descriptions

Association
is a structural relationship, specifying that objects of one thing are connected to objects of another

Dependency
is a using relationship, specifying that a change in the specification of one thing may affect another thing that uses it, but not in reverse.

Generalization
is a relationship between a general classifier (called the superclass or parent) and a more specific classifier (subclass or child).

Realization
You'll use realization in two circumstances: in the context of interfaces and in the context of collaborations. Use realization to specify the relationship between an interface and the class or component that provides an operation/service

Multiplicity

Graphically, a relationship is rendered as a path, with different kinds of lines used to distinguish the different relationships.
